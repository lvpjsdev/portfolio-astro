# Референсы и Источники Вдохновения

Этот документ содержит список источников, использованных для формирования концепции портфолио. Мы исключили стандартные шаблоны и взяли только самые яркие фичи из реальных работ профессиональных разработчиков.

---

## 1. Ключевые примеры портфолио (Основной источник фич)

*Источник: Статья "Top 23 Web Developer Portfolio Examples to Inspire Your Own" (WeAreDevelopers)*
**Ссылка:** https://www.wearedevelopers.com/en/magazine/web-developer-portfolio-examples

### Dustin Brett (Interactive & WebGL)
*   **Что найдено:** Портфолио описывается как "мастер-класс возможностей веб-разработки". Использует WebAssembly, WebGL и работает полностью на стороне клиента. Добавлены звуковые эффекты и ностальгический дизайн.
*   **Что взято для проекта:**
    *   Идея использования **WebGL / Three.js** на фоне (Hero Section).
    *   Концепция "погружения" — сайт не просто страница, а опыт (experience).

### Tamal Sen (IDE Aesthetic)
*   **Что найдено:** Темная тема, имитирующая интерфейс IDE (Integrated Development Environment). Фон украшен фрагментами кода.
*   **Что взято для проекта:**
    *   **Визуальный стиль:** Темная тема по умолчанию, моноширинные шрифты для заголовков, "компьютерная" эстетика.
    *   **Декор:** Идея использовать блоки кода как декоративные элементы.

### Brice Clain (Animated Storytelling)
*   **Что найдено:** Сайт наполнен анимациями, которые "проводят" вас через историю создания проектов. Ощущение личного разговора.
*   **Что взято для проекта:**
    *   **Story-driven Case Study:** Секция главного проекта строится не как галерея картинок, а как повествование (Problem -> Solution -> Result).
    *   Анимации появления элементов при скролле.

### Mason Wong (Retro & Holographic)
*   **Что найдено:** Дерзкий ретро-дизайн с голографическими элементами и плавными переходами.
*   **Что взято для проекта:**
    *   Использование **ярких градиентов и "голографических" акцентов** (неоновый зеленый + фиолетовый).
    *   Плавные hover-эффекты на карточках.

### Constance Souville (Editorial Minimalism)
*   **Что найдено:** Элегантный минимализм в стиле журнальной верстки. Сетка, типографика.
*   **Что взято для проекта:**
    *   **Лейаут сетки проектов (Grid Layout):** Отказ от стандартных квадратов в пользу более "журнальной" раскладки.
    *   Фокус на типографике и "воздухе".

## 2. Структурные паттерны и UI

*Источник: Статья "32 Examples Of Best Web Developer Portfolios For Design Inspiration" (MagicUI / SillyBits)*
**Ссылка:** https://magicui.design/blog/best-web-developer-portfolios

### Olaolu Olawuyi (Bright & Service-focused)
*   **Что найдено:** Яркие цвета, четкое изложение услуг сразу на первом экране.
*   **Что взято для проекта:**
    *   Идея **краткого интро (Intro Section)** сразу после Hero: четкое позиционирование ("Кто я" и "Что делаю").
    *   Использование ярких цветовых акцентов на темном фоне.

### Adham Dannaway (Split Personality)
*   **Что найдено:** Дизайн, демонстрирующий сочетание навыков дизайна и кода (часто через разделение экрана или графику).
*   **Что взято для проекта:**
    *   Баланс между визуальной красотой и технической частью (показываем и интерфейс, и код).

### Структура "Case Study"
*   **Что найдено в статье:** Рекомендация сопровождать каждый проект детальным описанием (Goals, Role, Tech Stack).
*   **Что взято для проекта:**
    *   Обязательное наличие **метрик и описания процесса** внутри карточки проекта, а не просто ссылка на GitHub.

## 3. Философия дизайна и подход

*Источник: Статья "10 Minimal Portfolio Examples for Web Developers Who Aren't Good at Design" (Scrimba)*
**Ссылка:** https://scrimba.com/articles/minimal-web-developer-portfolio-examples

### Принцип "Content First"
*   **Что найдено:** Дизайн не должен мешать контенту. Избегайте перегруженности.
*   **Что взято для проекта:**
    *   Решение использовать **CSS Modules** для локализации стилей и избежания CSS-хаоса.
    *   Отказ от тяжелых UI-кит (типа Bootstrap/MUI) в пользу кастомных, легких компонентов.
    *   Совет из комментариев: "Портфолио — это портал к вашей работе, а не сама работа" (не переусложнять).
